# Security Policy

## Supported Versions

Security fixes are applied to the current `develop` branch and the latest
published release when practical. Older releases may not receive patches.

## Reporting A Vulnerability

Please do not disclose vulnerabilities in a public issue, discussion, pull
request, or chat transcript.

Report privately to the repository maintainer through the contact method listed
on the project's GitHub profile. Include:

- a clear description of the issue and its potential impact;
- affected version, operating system, and configuration details;
- minimal reproduction steps or a proof of concept;
- suggested mitigation, if available.

Do not include real API keys, access tokens, private provider transcripts, or
personal data. We will acknowledge a valid report, investigate it privately,
and coordinate a fix and disclosure timeline with the reporter.

## Scope Notes

MindCraft Agent integrates local Claude Code and Codex tooling. Provider account
security, credentials, and upstream service availability remain the
responsibility of their respective providers. Reports are in scope when they
involve MindCraft Agent code, packaged application behavior, or its local data
handling boundaries.
